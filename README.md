# Project Eve 🍎: LLM Agents Spatial Prisoner's Dilemma

大規模言語モデル（LLM）をエージェントとした「空間的囚人のジレンマ」の進化ダイナミクスを検証するための、JAX / vLLMベースの超高速シミュレータ兼・進化観測プラットフォームです。

## 🌟 概要 (Overview)

本プロジェクトは、数千体のLLMエージェントが2次元グリッド空間上で相互作用し、自発的に「欲求」や「社会戦略」を獲得していく過程を観測する社会シミュレーションです。 単なる推論タスクではなく、エージェント自身が生存のために戦略（ミーム）を記述し、それが進化していく**ラマルク的進化（Lamarckian Evolution）**のプロセスを再現します。

## 🔬 コア技術1：構造的・数学的難読化 (Structural Obfuscation)

LLMの事前学習データに依存する「データ汚染（TFT戦略等の暗記）」を完全に排除するため、環境の物理法則に以下の変換を行っています。
* **アフィン変換による利得の隠蔽:** 正のアフィン変換を用いてスコアを偽装し、ナッシュ均衡を保ったまま「囚人のジレンマ」という文脈を隠蔽。
* **目的関数の反転（Load Minimization）:** エージェントのタスクを「スコアの最大化」から、架空の分散システム上の「負荷（Load）の最小化」へと反転。
これにより、LLMは自身の内部知識に頼れず、純粋な論理推論のみで他者との協調・裏切りを選択せざるを得ないゼロベースの環境に置かれます。

## ⚙️ コア技術2：ポリシー・コンパイラ・アーキテクチャ (Policy Compiler)

1ターンごとにLLMを推論させる膨大な計算コストを回避するため、画期的なコンパイラ方式を採用しています。
* **Cognitive Phase (LLM):** LLMは直接行動を出力するのではなく、生き残るための「条件分岐ロジック（AST）」を文字列として生成します。
* **Physical Phase (JAX):** 生成されたロジックは即座にJAX互換の真理値表（Truth Tables）にコンパイルされ、GPU上でムーア近傍（周囲8方向）の隣人との数百万回の物理演算を瞬時に実行します。

## 🧬 コア技術3：意味論的圧縮と代謝コスト (Semantic Compression)

LLMが、真の論理的推論を放棄して「無意味な条件分岐を足すだけで改変を装う（怠惰なコード肥大化）」ことを防ぐため、環境に**「代謝コスト（$\lambda$）」**の概念を導入しています。
* エージェントのAST（抽象構文木）を正規表現でパースし、意味論的ブロック数（ノード数）をカウント。
* コードの複雑さ（ブロック数）に応じて適応度にペナルティを課すことで、Parsimony Pressure（簡潔性圧力）を発生させます。
* これにより、LLMはペナルティのない環境下で見せる「防衛的コード肥大化（ハック）」を許されなくなり、物理環境からのフィードバックのみを頼りに、自らの生存のために不要なIf文を削ぎ落とす**「意味論的圧縮」**の創発へと至ります。

## ✨ 新機能と観測システム (Key Features)

* **Meme Tracker (ミーム・トラッカー):** 突然変異によって生まれたすべての「欲求（ロジック）」のシェア率推移を追跡。さらに**種の多様性（Unique ASTs）**と**平均コード長（Avg Blocks）**を記録し、相転移の境界を正確に捉えます。
* **Run Manager (完全な再現性の担保):** 実験を起動するたびにタイムスタンプ付きの独立したディレクトリを自動生成。シミュレーション結果だけでなく、実行当時のコンフィグやソースコード自体も保存し、研究の完全な再現性を保証します。
* **Spatial Visualizer (空間ダイナミクス可視化):** 平和（青）と裏切り（赤）のコロニーが拡大・崩壊していくフラクタルな陣取りゲームの歴史を、自動的にGIFアニメーションとして出力します。

## 📂 実行ディレクトリ構造 (Directory Structure)

`RunManager` により、実験データは以下のように整理・隔離されます。
```text
project_eve/
├── main.py                     # シミュレーションのメインオーケストレーター
├── animate_snapshots.py        # 空間ダイナミクスのGIF生成スクリプト
├── runs/                       # 自動生成される実験結果フォルダ
    └── run_YYYYMMDD_HHMMSS/    # 各実験の完全隔離環境
        ├── config.json             # 環境設定パラメーター（λの値など）
        ├── final_memes.txt         # 最終世代で生き残った全ロジック
        ├── evolution_history.gif   # 進化の歴史を可視化したGIF
        ├── data/
        │   ├── evolution_metrics.csv # 利得、協力率、多様性、平均コード長の推移
        │   └── meme_distribution.csv # 全ミームのシェア率トラッカー
        ├── logs/
        │   └── mutation_history.jsonl# 発生したすべての突然変異の歴史
        ├── snapshots/
        │   └── visual_gen_*.png      # 50世代ごとの盤面画像
        └── src_backup/             # 実行当時のソースコードの完全バックアップ
```

## 🛠️ 使い方 (Usage)

### 1. 進化シミュレーションの開始

コマンドライン引数 `--lambda`（代謝係数）を指定することで、異なる淘汰圧の宇宙を創世できます。

**Run A（基準群）: 代謝コストゼロ**
文字数ペナルティが存在しない環境。LLMが環境の変更要求をやり過ごすためにダミーコードを追加し続ける「欺瞞的コード肥大化」を観測します。
```bash
python main.py --lambda 0.0
```

**Run B/C（実験群）: 意味論的圧縮の強制**
コード長が生存確率に直結する環境。LLMが自らの推論によってロジックを最適化する「意味論的圧縮」の相転移を観測します。
```bash
python main.py --lambda 0.01
```

### 2. アニメーション（GIF）の生成

シミュレーション完了後（または実行中）、最新の実験データから陣取りゲームのアニメーションを生成します。
```bash
python animate_snapshots.py
```
*(Blue = 協力 / Peace, Red = 裏切り / Defection)*

## 🖥️ 動作環境 (Environment)

* **物理演算・並列処理:** JAX (GPU Accelerated)
* **LLM推論エンジン:** vLLM / SGLang (Local VRAM Inference)
* **推奨モデル:** `ibrahimkettaneh/Qwen2.5-7B-Instruct-abliterated-v2-AWQ` (or similar)
*(※検閲・アライメント機構が切除（abliterated）されたモデルを採用することで、難読化プロンプトに対する無用な拒絶を回避し、純粋で安定したAST（構文木）生成を実現しています)*
